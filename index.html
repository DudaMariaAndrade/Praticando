Questionário

Questão 1: Explique qual é a função de um servidor web em um ambiente de hospedagem e cite pelo menos um exemplo.

Resposta: Um servidor web é um software que “entrega” o conteúdo de um site para o navegador do usuário. O servidor Apache é um servidor web popular, que pode ser usado para hospedar sites na internet. Um dos mais populares e usados no mundo. Código aberto .
Questão 2: Qual a principal característica do NGINX e por que ele é muito utilizado em projetos de alto desempenho? .

Resposta: O NGINX é conhecido pela sua alta performance em lidar com muitas requisições ao mesmo tempo. Ele é muito eficiente porque usa um modelo de processamento assíncrono, o que significa que não precisa esperar que uma tarefa termine para começar outra. Ele é muito utilizado em projetos que precisam de velocidade e baixa latência, como sites de alto tráfego.
Questão 3: Diferencie os protocolos HTTP e HTTPS. Por que o HTTPS é recomendado para aplicações modernas?

Resposta: HTTP: protocolo para transferência de páginas web (sem criptografia), ou seja, comunicação entre cliente (navegador) e servidor. HTTPS: versão segura com criptografia SSL/TLS. Portando a diferença que um garante que os dados trocados sejam protegidos e o outo não
Questão 4:Qual é a função das portas em uma rede de computadores e qual a importância da porta 443?

Resposta: As portas são identificadores numéricos usados para direcionar o tráfego de dados para o serviço correto dentro de um servidor. Cada tipo de serviço usa uma porta específica. Por exemplo, o HTTP geralmente usa a porta 80, enquanto o HTTPS usa a porta 443. Porta 443 é a porta usada para conexões seguras via HTTPS. Sua importância vem da sua segurança.
Questão 5:Defina o conceito de endereço IP e explique sua importância em redes de computadores.

Resposta: número identificador de um dispositivo na rede. Ele é como o "endereço de uma casa" na internet. Sem ele, não seria possível localizar e se comunicar com outros dispositivos ou servidores na rede.
Questão 6: Explique o que é hospedagem VPS e em quais situações ela é mais indicada.

Resposta: É servidor virtual dedicado com mais controle e desempenho. Ele compartilha recursos físicos com outros servidores. É mais flexível e seguro que a hospedagem compartilhada, pois você tem mais controle sobre o servidor. E é indicado quando um projeto está crescendo e precisa de mais recursos e segurança do que uma hospedagem compartilhada oferece, mas você não quer pagar por um servidor dedicado completo.
Questão 7: Descreva as vantagens do servidor web Caddy em relação a outros servidores tradicionais.

Resposta: É um Servidor moderno com HTTPS automático e configuração simples. Tem como grande vantagem a configuração automática de SSL (HTTPS). Ele é muito fácil de usar e tem suporte nativo para HTTPS, o que facilita a implementação de sites seguros sem precisar configurar manualmente o certificado SSL.
Questão 8:Qual servidor de aplicação é mais indicado para aplicações desenvolvidas na plataforma .NET (C#)? Explique o motivo.

Resposta: O servidor de aplicação mais indicado para .NET (C#) é o Kestrel, que é um servidor web embutido no ASP.NET Core. Ele é rápido e eficiente, projetado para trabalhar com aplicações web desenvolvidas com essa tecnologia.
Questão 9: Descreva o funcionamento do modelo Cliente-Servidor no contexto de sistemas web.

Resposta: No modelo Cliente-Servidor, o "cliente" é o usuário (geralmente um navegador), e o "servidor" é o sistema que fornece os dados. O cliente faz uma solicitação (por exemplo, acessar uma página web) e o servidor processa e responde com a informação solicitada (por exemplo, o HTML da página).
Questão 10: Explique o que é o Apache Tomcat e para qual tipo de aplicação ele é mais indicado.

Resposta: O Apache Tomcat é um servidor de aplicação que executa aplicações desenvolvidas em Java. Ele é mais indicado para aplicações web baseadas em Java EE (Enterprise Edition), como aquelas que usam Servlets e JSP (JavaServer Pages). Muito utilizado em aplicações java.
Questão 11: Diferencie, com suas palavras, um servidor web de um servidor de aplicação, destacando suas funções principais.

Resposta: Servidor Web: entrega arquivos estáticos como HTML, CSS e imagens. Servidor de Aplicação: executa lógica da aplicação e gera respostas dinâmicas.
Questão 12: Defina o conceito de domínio na internet e sua relação com os endereços IP.

Resposta:Domínio: nome que representa um IP. Ou seja, para acessar um site. Por trás do domínio, existe um endereço IP que é o identificador real do servidor.
Questão 13: Quais as principais vantagens da hospedagem em nuvem em comparação com a hospedagem local?

Resposta: A hospedagem em nuvem oferece escalabilidade (facilidade para aumentar ou diminuir os recursos conforme necessário) e maior disponibilidade (os servidores estão distribuídos em diferentes locais). Em comparação, a hospedagem local exige que você gerencie o hardware e a infraestrutura, o que pode ser mais caro e menos flexível.
Questão 14: Cite dois exemplos de servidores de aplicação e as linguagens/plataformas que eles suportam.

Resposta: Tomcat -> Suporta aplicações Java; Node.js -> Suporta aplicações JavaScript/TypeScript.
Questão 15: Por que é essencial a utilização de HTTPS em sites e sistemas atualmente? Quais riscos o HTTP apresenta?

Resposta:O uso de HTTPS é importante para proteger dados sensíveis trocados entre o cliente e o servidor. HTTP, sem a criptografia, é vulnerável a ataques como man-in-the-middle, onde alguém pode interceptar os dados.
Questão 16: Dê um exemplo de situação onde o uso de hospedagem VPS seria mais vantajoso do que uma hospedagem compartilhada.

Resposta: Se você tem um site que começa a receber mais tráfego e precisa de mais recursos do que uma hospedagem compartilhada pode oferecer, a VPS seria mais vantajosa, pois você terá mais controle e recursos dedicados.
Questão 17: Diferencie os conceitos de IP e porta em uma rede, explicando suas funções.

Resposta: IP: Identifica o dispositivo na rede. Porta: Identifica o serviço específico no dispositivo (como o servidor web, servidor de e-mail etc.).
Questão 18: Qual a importância de um servidor web conseguir lidar com conexões via HTTPS?

Resposta: O servidor web precisa lidar com HTTPS para garantir que a comunicação seja segura e que os dados dos usuários (como senhas e informações bancárias) não sejam interceptados.
Questão 19: Cite dois exemplos de servidores web e explique suas principais características.

Resposta: Microsoft IIS: Servidor web da Microsoft, integrado ao Windows Server. LiteSpeed: Focado em performance e compatível com Apache.
Questão 20: Defina o que é uma hospedagem compartilhada e em quais casos ela costuma ser utilizada.

Resposta: Hospedagem Compartilhada: vários sites no mesmo servidor físico, com baixo custo. É mais barata, mas pode ser menos eficiente para sites com muito tráfego. Ideal para sites pequenos ou pessoais.
Questão 21:< Explique a função de um servidor de aplicação e a diferença em relação a servidores que apenas servem arquivos estáticos.

Resposta: O servidor de aplicação processa dados dinâmicos e executa código (como um sistema de login ou interação com um banco de dados), enquanto os servidores de arquivos estáticos apenas entregam arquivos como HTML, CSS e imagens sem fazer processamento.
Questão 22: Cite um servidor de aplicação usado para projetos em Python e descreva como ele funciona.

Resposta:O Gunicorn é um servidor de aplicação popular para projetos em Python. Ele serve para rodar aplicações web, especialmente aquelas feitas com Flask ou Django.
Questão 23: O que é o WildFly (ex-JBoss) e para qual tecnologia ele é indicado?

Resposta: O WildFly é um servidor de aplicação Java que é usado para rodar aplicações Java EE (Enterprise Edition). Ele é indicado para aplicações mais robustas e empresariais.
Questão 24: Qual a função da porta 21 em redes de computadores e para qual serviço ela é utilizada?

Resposta: A porta 21 é usada pelo serviço FTP (File Transfer Protocol), que serve para transferir arquivos entre um computador e um servidor.
Questão 25: Explique por que o servidor LiteSpeed é compatível com o Apache e quais as vantagens dessa compatibilidade.

Resposta:O LiteSpeed é compatível com o Apache porque pode usar as mesmas configurações e arquivos .htaccess. A vantagem é que ele oferece uma performance superior e segurança aprimorada, mantendo a facilidade de configuração do Apache.
Questão 26: O que é o Microsoft IIS e em quais situações ele costuma ser adotado?

Resposta: O Microsoft IIS (Internet Information Services) é um servidor web para o sistema operacional Windows. Ele é mais usado em ambientes empresariais que já utilizam produtos Microsoft, como ASP.NET.
Questão 27: Explique para que serve o Node.js e em quais tipos de aplicações ele é mais utilizado.

Resposta: O Node.js é uma plataforma de desenvolvimento que permite rodar JavaScript no servidor. Ele é muito utilizado em aplicações em tempo real, como chats, APIs e aplicações de streaming.
Questão 28: Descreva a função do servidor GlassFish e para qual linguagem ele é destinado.

Resposta: O GlassFish é um servidor de aplicação que implementa a plataforma Java EE (atualmente conhecida como Jakarta EE). Ele é projetado para rodar aplicações empresariais baseadas em Java, como sistemas complexos de gerenciamento de dados e transações. O GlassFish suporta tecnologias como Servlets, JSP, EJB e JPA, entre outras.
Questão 29: O que são Puma e Unicorn e em quais projetos eles são normalmente utilizados?

Resposta: Puma e Unicorn são servidores de aplicação para Ruby on Rails. Puma é multithreaded, moderno e eficiente. Unicorn é multiprocessado, mais simples e oferece isolamento entre processos. Ambos são usados em aplicações Ruby, mas o Puma é mais popular por ser mais rápido e escalável.
Questão 30: Qual a importância da escolha adequada entre hospedagem local, compartilhada, VPS e nuvem no planejamento de um projeto web?

Resposta: A escolha do tipo de hospedagem é crucial no planejamento de um projeto web, pois afeta diretamente a performance, a segurança e o custo do seu sistema. Cada tipo de hospedagem tem vantagens e desvantagens dependendo do porte e das necessidades do projeto.
